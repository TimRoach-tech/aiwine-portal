What this does & doesn't do. It only ever reads — it never creates a winery, an order, or writes any data. It can't see inside row-level security (that's the point of RLS), so a few checks confirm a table or function exists and is reachable rather than counting rows. The one check that proves your data is safe — signing in as Winery A and confirming you can't see Winery B — still has to be done by hand with two real logins.

How to read it. Green = live and working. Red = missing or broken (the detail line says what). Amber = couldn't determine automatically — check it manually. The headline banner tells you, in one line, whether the winery sign-in is ready to deploy.
